Using discrete probability models for binary events

We use Poisson and Binomial distributions as models for binary outcomes.

Poisson Model for rare events

In the HIV model above, suppose we want to find the probability of seeing 3 mutations in a sequence of length 10,000?

dpois(x=3,lambda=5)
## [1] 0.14

There is a formula: \[ P(X=k)= \frac{\lambda^ke^{-\lambda}}{k!} \mbox{ For }\lambda=5, \mbox{ we get }P(X=3)=\frac{5^3\times e^{-5}}{3\times 2}.\]

Binomial Success Counts

When we have a binary outcome (success/failure), (CpG/NonCpG), (M/F), (Y=Pyraminidine,R=Purine), (Diseased/Healthy), True/False.

We can model it as a simple random variable with probability of success \(p\) and failure \(1-p\).

SSSSSFSSSSFFFSF is summarized as (#Success=10,#Failures=5).

The number of successes is said to have a Binomial distribution with parameters \(prob=0.3\) and \(size=15\).

Multinomial Distribution: the formula

Here is the formula the programs use to compute the probability of a multinomial vector of counts \((x_1,\ldots,x_m)\) being observed: \[ \begin{eqnarray*} P(x_1,x_2,\ldots,x_m | p_1,\ldots,p_m)&=&\frac{n!}{\prod x_i!} \prod p_i^{x_i}\\ &=&{{n}\choose{x_1,x_2, x_m}} p_1^{x_1}p_2^{x_2} \cdots p_m^{x_m} \end{eqnarray*} \]

  for ( b in 1:B){

Poisson

When an event is quite rare, like a mutation, the number of times it occurs follows a Poisson distribution.

If we only look at one trial, most of the time, we wont see anything.

For 1000 positions of the RT gene, and each mutation occurs independently at rate 0.02 then we would expect to see 20 mutations (1000 \(\times\) 0.02= \(n \times p\)).

In general if an event has probability p and and we have n trials, this number is random with a Poisson distribution with mean parameter \(\lambda\) equal to np.

sum(rpois(1000,lambda=0.02))
## [1] 27

If you do this quite a few times, youll how the values you get differ, we get a distribution of values centered around 20 or so.

Using probabilistic (generative) models for Epitope Detection

When testing certain pharmaceutical compounds it is important to detect proteins that provoke an allergic reaction, the sites that are responsible for such reactions are called epitopes.

The technical definition of an Epitope is: A specific potion of a macromolecular antigen to which an antibody binds.

An antibody http://ghr.nlm.nih.gov/ghr/glossary/antibody is type of protein made by certain white blood cells in response to the foreign substance which is called the antigen.

Each antibody can bind to only a specific antigen.

The purpose of this binding is to help destroy the antigen.

Some antibodies destroy antigens directly.

An epitope (or antigenic determinant), is the part of an antigen that is recognized by the immune system

We see a surprising spike and have to answer the question:

What are the chances of seeing a value as large as 7?

If we only look for the probability of seeing a number as big as 7 when considering one Poisson(\(0.5\)) random variable. \[ P(X\geq 7)= 1-P(X\leq 6)=1-F_{Pois}(6)=1-ppois(6,0.5) \] Ill call this number \(\epsilon\), how big is it?

##Here are the cumulative function values for 0 to 6
round(ppois(0:6,0.5),5)
## [1] 0.607 0.910 0.986 0.998 1.000 1.000 1.000
##What are the chances of seeing a value strictly bigger than 6?
1-ppois(6,0.5)
## [1] 1e-06

Extreme Value Theory (maximum value for Poisson)

This would actually not be the correct computation, in fact we looked at all 100 positions and 7 was chosen because it was the maximum, so we have to ask ourselves what are the chances of seeing a number as large in 100 trials.

We use theory here. We order the values \(x_1,x_2,\ldots x_n\) renaming them \(x_{(1)},x_{(2)},x_{(3)}\ldots x_{(100)}\) so that \(x_{(100)}\) denotes the maximum of the counts over the 100 positions.

For the largest to be as large as 7 is the (opposite) of having all 100 counts be smaller or equal to 6.

\[P(x_{(100)}\geq 7)=1-P(x_{(100)} \leq 6)=1- \Pi_{i=1}^{100} P(x_i \leq 6 )\] (supposing independence) \[\Pi_{i=1}^n P(x_i < 7)=\left(\sum_{k=0}^{6} \frac{e^{-\lambda} \lambda^k}{k!}\right)^{100}= \left(1-\sum_{k=7}^{\infty} \frac{e^{-\lambda} \lambda^k}{k!}\right)^{100}\] (supposing identically dist. Poisson)

Estimating small numbers

\(\epsilon= \sum_{k=7}^{\infty} \frac{e^{-\lambda} \lambda^k}{k!}=P(x_i\geq 7)=1.00238.10^{-6}\).

Using this we can compute

\[\Pi_{i=1}^{100} P(x_i < 7)=(1-1.00238 .10^{-6})^{100}\]

We use the approximation \(\log(1-\epsilon)\simeq -\epsilon\).

\[\left(1-\epsilon \right)^{100} =\exp ( 100 \log (1-\epsilon))=\exp({- 100 \epsilon}) =e^{-100 P(x_i\geq 7)}=e^{-100\times 10^{-6}}=e^{-10^{-4}} \]

So that \[P(x_{(100)}\geq 7)= 1-exp(-10^{-4}) =9.9995\; .\; 10^{-5}\]

We could also do this by simulation, or what is known as the Monte Carlo method (it involves throwing a lot of dies).

Why is this better than by simulation (Monte Carlo) ?

Do not run this as it takes quite a lot of time and memory (on a 4GB machine this computation took about an hour, whereas the small analytic approximation above took 2 minutes)

M100=rpois(1000000000,0.5)
table(M100)
Matrix100=matrix(M100,ncol=100)
vecmaxes=apply(Matrix100,1,max)
table(vecmaxes)

giving an approximation of \(9.48. 10^{-5}\) for \(P(X_{max}\geq 7)\) and \(3.10^{-7}\) for \(P(X_{max}\geq 9)\).

We can however prove that a probability is smaller than 0.000001 for instance by doing just \(10^6\) computations.

million=rpois(1000000,0.5)
mat100=matrix(million,ncol=100)
maxes100=apply(mat100,1,max)
table(maxes100)

## maxes100
##   1    2    3    4    5    6 
##   2 2338 6053 1432  163   12

We can only conclude that \(P(max \geq 7)<10^{-4}\). So although it is doable to find small probabilities by simulation it is not always optimal.

Important things we learnt today

  • We have learned several ways to model count data, the Poisson, Binomial and Multinomials.
  • We learnt how to generate random discrete data using the specialized R functions tailored for each type of distribution.
  • The epitope example showed us how to use a probability model to compute a probability under a parametric model when the parameter was known. This probability was in fact so small it would even have been hard to compute by Monte Carlo simulation.
  • P values are calculated from probabilities under certain assumptions. We will use probability distributions and probabilistic models to evaluate many of our results and compute both p-values and probabilities of our hypotheses being verified by the data.

R note about known distributions

All known distributions can be used to simulate data using the functions that start rXXXX where XXXX could be pois, binom, multinom. If we need a theoretical computation of a probability under one of these models, we would use the functions dXXXX, such as dpois.

This also works for distributions that are not counts, such the Gaussian ( rnorm, dnorm), the \(\chi^2\) ( rchisq, dchisq) or the exponential ( rexp, dexp).
